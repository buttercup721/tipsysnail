# Snail Terrarium MVP Plan

## 1. Game Direction

### One-line pitch
`Hammy Home`의 강점인 "꾸미기 + 가벼운 상호작용 + 관찰" 구조를 가져오되, 달팽이 특유의 느림, 촉촉함, 벽 타기, 점액 자국, 작은 생태계를 중심 재미로 바꾼 웹/PWA 육성 게임.

### Core fantasy
- 플레이어는 "달팽이를 관리하는 주인"이라기보다 "작은 테라리움을 가꾸는 큐레이터"가 된다.
- 달팽이는 장난감처럼 빠르게 반응하는 펫이 아니라, 관찰할수록 귀엽고 디테일이 보이는 생물로 표현한다.
- 실패 페널티보다 "잘 돌볼수록 더 예쁜 장면을 자주 본다"는 보상 구조를 우선한다.

### Reference adaptation from Hammy Home
- 유지할 점:
  - 귀엽고 편한 감성
  - 꾸미기 중심 플레이
  - 짧은 상호작용 버튼
  - 로컬 저장
  - 스냅샷 공유
- 달팽이에 맞게 바꿀 점:
  - 햄스터의 빠른 장난감 액션 대신 느린 이동과 관찰 중심 연출
  - 케이지 대신 촉촉한 테라리움
  - 바퀴, 튜브 대신 잎, 이끼, 버섯, 나무 조각, 돌, 유리벽
  - 달팽이 고유 포인트로 점액 자국, 습도 반응, 벽 타기, 껍데기 무늬 추가

## 2. MVP Scope

### Platform
- 웹
- PWA 설치 지원
- 모바일 세로 화면 우선
- 데스크톱 브라우저 대응

### Content scope
- 달팽이 5종
- 스타터 테라리움 2종
- 소품 18개
- 버튼 4개
- 로컬 저장
- 스냅샷 공유

### Important assumption
사용자 요청에는 "버튼 4개"라고 되어 있지만 실제로는 `먹이`, `터치`, `교배` 3개만 명시되어 있다. MVP 진행을 위해 4번째 버튼은 `편집`으로 가정한다. 이 버튼이 있어야 소품 배치와 테라리움 꾸미기 흐름이 자연스럽다.

## 3. Core Game Loop

1. 플레이어가 테라리움에 들어간다.
2. 달팽이가 천천히 움직이고, 벽을 타고, 쉬고, 먹는다.
3. 플레이어는 `먹이`, `터치`, `교배`, `편집` 버튼으로 개입한다.
4. 테라리움을 꾸미고 달팽이 조합을 바꾸면서 더 귀엽고 희귀한 장면을 만든다.
5. 마음에 드는 순간을 스냅샷으로 저장하거나 공유한다.
6. 다음 접속 시 로컬 저장 데이터로 이어서 플레이한다.

## 4. MVP Feature Set

### A. Snail species (5)
각 종은 실제 생물 고증보다 "색감과 성격 차이"를 우선한 캐주얼 디자인으로 간다.

1. Garden Snail
   - 기본형
   - 갈색 껍데기, 안정적 성격
2. Amber Snail
   - 밝은 꿀색 껍데기
   - 먹이를 빨리 발견하는 편
3. Moss Snail
   - 초록빛 무늬
   - 이끼 소품 근처를 선호
4. Moon Snail
   - 옅은 회백색 껍데기
   - 밤 배경에서 더 자주 활동
5. Strawberry Snail
   - 분홍/적색 포인트
   - 희귀도 높은 수집형 포지션

### B. Starter terrariums (2)
1. Moss Jar
   - 둥근 유리병 스타일
   - 초보자용 기본 배치
   - 따뜻하고 아기자기한 분위기
2. Rain Window Tank
   - 넓은 직사각 테라리움
   - 유리벽 표현이 강조됨
   - 벽 타기와 점액 자국 연출에 유리

### C. Props (18)
소품은 장식성과 행동 유도성을 함께 가진다.

1. Moss Carpet
2. Fern Pot
3. Small Mushroom Cluster
4. Tall Mushroom Cluster
5. Flat River Stone
6. Pebble Pile
7. Bark Tunnel
8. Hollow Log
9. Leaf Arch
10. Lotus Bowl
11. Dew Dish
12. Tiny Signpost
13. Clay Pot Hideout
14. Mini Waterfall Rock
15. Hanging Vine
16. Sprout Patch
17. Acorn Lamp
18. Clover Patch

### D. Main action buttons (4)
1. 먹이
   - 상추 조각, 오이 조각 같은 단순 먹이 아이템이 테라리움 안에 떨어진다.
   - 주변 달팽이가 먹이 쪽으로 천천히 이동한다.
2. 터치
   - 달팽이를 쓰다듬거나 톡 건드리는 상호작용
   - 긍정 반응 시 눈이 움직이거나 몸을 살짝 뻗는다.
   - 과도한 연타는 피하고 짧은 쿨다운을 둔다.
3. 교배
   - 성체 달팽이 2마리를 선택해 알을 낳게 한다.
   - 일정 시간 후 부화
   - 부모 껍데기 색/무늬 조합을 일부 계승
   - 희귀 조합 수집의 핵심 루프
4. 편집
   - 소품 배치/회수/정렬
   - 달팽이 배치 확인
   - 배경과 테마 변경 진입

## 5. Systems

### Mood and condition
MVP에서는 복잡한 수치 관리를 줄이고, 보이는 상태 중심으로 간다.

- Hunger
  - 낮음 / 보통 / 높음 정도의 단순 상태
  - 먹이 버튼과 연결
- Comfort
  - 은신처, 장식 밀도, 선호 소품에 의해 상승
- Bond
  - 터치 상호작용 누적으로 소폭 상승
- Breeding readiness
  - 일정 시간 경과 후 교배 가능

플레이어에게 모든 수치를 숫자로 직접 노출하지 않고, 표정/행동/아이콘으로 부드럽게 전달하는 편이 좋다.

### Behavior system
- 바닥 이동
- 유리벽 타기
- 먹이 냄새를 맡고 접근
- 소품 아래 숨기
- 휴식
- 서로 마주치면 짧은 인사 애니메이션
- 이동 후 일정 시간 점액 자국 남기기

### Breeding system
- 달팽이 2마리 선택
- 같은 테라리움 내 성체만 가능
- 교배 후 알 1~3개 생성
- 알은 장식 근처나 흙 위에 배치
- 부화 시 부모 색상/패턴을 확률 혼합
- 희귀 품종 해금의 주요 수단

### Save system
- 브라우저 로컬 저장
- 저장 대상:
  - 보유 달팽이
  - 테라리움 배치
  - 해금한 소품
  - 교배 결과
  - 스냅샷 메타데이터

### Snapshot share
- 현재 테라리움 장면을 이미지로 저장
- 모바일 공유 시트 또는 다운로드 지원
- 달팽이 이름과 날짜를 얹는 단순 프레임 옵션 제공

## 6. Player Progression

### Short-term goals
- 달팽이 먹이 주기
- 예쁜 배치 만들기
- 교배 결과 확인하기

### Mid-term goals
- 5종 수집
- 희귀 껍데기 패턴 수집
- 소품 세트 완성

### Long-term feel
- 내 테라리움이 점점 살아 있는 작은 정원처럼 느껴진다.
- 매번 접속할 때 새로운 작은 장면이 생긴다.

## 7. Art and UX Direction

### Visual tone
- 포근함, 습기, 초록빛, 유리 반사, 부드러운 채도
- 너무 리얼한 생물 묘사보다 동글고 귀여운 실루엣
- 점액 자국은 혐오스럽지 않게 반짝이는 선으로 표현

### UI tone
- 큰 버튼
- 직관적인 아이콘
- 텍스트는 짧고 부드럽게
- HUD는 최소화하고 장면 감상 영역을 넓게 확보

### Motion direction
- 빠른 반응보다 작은 디테일이 중요
- 먹을 때 입 움직임
- 벽을 탈 때 더듬이 흔들림
- 터치 시 몸이 살짝 움찔
- 물기 표현용 작은 반짝임

## 8. Screen Structure

### 1. Home / Main terrarium
- 현재 테라리움 표시
- 달팽이 행동 관찰
- 하단 4버튼 배치
- 상단에 이름, 설정, 스냅샷

### 2. Edit mode
- 소품 목록
- 드래그 배치
- 회전 또는 좌우 반전은 선택 사항
- 레이어 자동 정렬

### 3. Breeding modal
- 부모 달팽이 2마리 선택
- 예상 결과는 실루엣 또는 확률 힌트만 제공
- 알 생성 후 부화 대기 상태 표시

### 4. Collection book
- 보유 달팽이 목록
- 껍데기 패턴 컬렉션
- 간단한 소개 문구

## 9. Recommended MVP Build Order

1. 메인 테라리움 화면
2. 달팽이 이동/벽 타기 기본 애니메이션
3. 편집 모드와 소품 배치
4. 먹이 상호작용
5. 터치 반응
6. 교배와 알/부화 시스템
7. 로컬 저장
8. 스냅샷 공유
9. PWA 설치 지원

## 10. Technical Recommendation

### Frontend
- React + TypeScript
- Canvas 또는 DOM/SVG 혼합
- 작은 규모 MVP라면 DOM/SVG도 충분히 가능

### State
- localStorage 또는 IndexedDB
- 배치 데이터와 달팽이 개체 수가 늘어날 가능성을 보면 IndexedDB가 더 안전

### Snapshot
- DOM 기반이면 `html-to-image` 계열 접근이 편함
- Canvas 기반이면 직접 export가 쉬움

### PWA
- 오프라인 기본 진입
- 아이콘, 스플래시, 설치 프롬프트 지원

## 11. MVP Success Criteria

- 첫 접속 1분 안에 달팽이가 귀엽다고 느껴진다.
- 편집 모드 없이도 기본 관찰 재미가 있다.
- 먹이와 터치의 반응이 충분히 사랑스럽다.
- 교배 결과를 기다리는 동기가 생긴다.
- 스냅샷으로 자랑하고 싶어진다.

## 12. Post-MVP Expansion Ideas

- 날씨 시스템
- 분무 버튼 추가
- 밤/낮 연출
- 희귀 이벤트 달팽이
- 달팽이 이름표 꾸미기
- 테마 팩 확장
- 친구 초대용 공유 링크

## 13. Final Positioning

이 MVP는 "달팽이를 죽지 않게 관리하는 게임"이 아니라, "작고 느린 생명체가 사는 촉촉한 정원을 꾸미고 관찰하는 게임"으로 포지셔닝한다. `Hammy Home`의 접근성은 유지하되, 달팽이만의 느림과 질감을 전면에 내세우는 것이 가장 중요한 차별점이다.
